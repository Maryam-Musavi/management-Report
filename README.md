# 📊 دستیار هوشمند اوراق گام (نسخه SQLite)

پروژه گزارش‌گیری هوشمند از جدول مشتریان اوراق گام با استفاده از **Llama 3.1:8b**، **Streamlit** و **SQLite**

---

## 📁 ساختار پروژه

```
gam_chatbot/
│
├── streamlit_app.py      # فایل اصلی Streamlit (UI)
├── gam_constants.py      # ثابت‌ها، تنظیمات، System Prompt
├── gam_functions.py      # توابع تحلیل داده و ساخت prompt
├── gam_database.py       # ماژول اتصال و خواندن/نوشتن SQLite
├── build_database.py     # اسکریپت مستقل تبدیل فایل خام → پایگاه داده تمیز
├── dt_ollama.py          # ماژول ارتباط با Ollama
├── requirements.txt      # پکیج‌های مورد نیاز
├── README.md             # این فایل
│
└── data/
    └── gam_customers.db  # پایگاه داده SQLite (آماده و تمیز)
```

---

## ⚙️ چرا SQLite به‌جای Excel؟

| | Excel | SQLite |
|---|---|---|
| پایداری داده بین اجراها | نیاز به آپلود مجدد هر بار | خودکار، دائمی روی دیسک |
| فیلتر/شمارش دقیق | پردازش کامل در پانداس | با SQL هم ممکن است |
| ساختار و اعتبارسنجی نوع داده | آزاد و مستعد خطا | ستون‌بندی مشخص |
| اتصال به سیستم‌های دیگر (مثلاً CRM) | دشوار | ساده (فایل .db استاندارد) |

برای این پروژه (~۵۰ رکورد)، تفاوت سرعت محسوس نیست؛ مزیت اصلی SQLite **پایداری** و **تمیزی ساختار** است.

## 🚀 نحوه اجرا

```bash
pip install -r requirements.txt
ollama pull gemma3:1b
ollama serve
streamlit run ./streamlit_app.py
```

پایگاه داده تمیز (`data/gam_customers.db`) از قبل آماده و در پروژه موجود است — نیازی به هیچ آپلودی برای شروع نیست.

## 🔄 وقتی گزارش/فایل جدیدی دریافت کردید

هر بار که فایل خام جدیدی (چه `.sqlite` چه `.xlsx`) از سامانه دیگری دریافت کردید، این دستور را اجرا کنید:

```bash
python build_database.py path/to/new_raw_file.sqlite
```

این اسکریپت به‌طور خودکار:
- اگر فایل ساختار ساده (یک جدول) داشته باشد → مستقیماً پاکسازی و ذخیره می‌کند
- اگر فایل ساختار دوتایی (دو جدول چسبیده، مثل نمونه اولیه این پروژه) داشته باشد →
  دو نیمه را جدا می‌کند، رکوردهای واقعاً یکسان را با احتیاط ادغام می‌کند (بدون
  از دست دادن داده‌های متضاد)، و رکوردهای جدید/متفاوت را به‌عنوان مشتری جداگانه
  اضافه می‌کند.

همچنین می‌توانید از **سایدبار برنامه** یک فایل `.sqlite`/`.db` (با همان ساختار
ستونی پروژه) را مستقیماً بارگذاری کنید تا جایگزین داده فعلی شود.

## 🏗️ معماری

```
کاربر → Streamlit UI → gam_functions.py → gam_database.py → SQLite (data/gam_customers.db)
                              ↓                                     ↑
                         dt_ollama.py                    build_database.py (یک‌بار، هنگام دریافت فایل جدید)
                              ↓
                      Ollama (llama3.1:8b)
```

### نحوه کار:
- سوال کاربر ابتدا در Python تحلیل و فیلتر می‌شود (شمارش/فیلتر دقیق، بدون خطای مدل)
- فقط داده‌های مرتبط + آمار دقیق به‌عنوان "Python pre-computation" به مدل ارسال می‌شود
- مدل فقط مسئول نوشتن پاسخ طبیعی فارسی است، نه شمارش یا فیلتر کردن
- تاریخچه گفتگو در `session_state` نگهداری می‌شود

## 📝 نکات مهم درباره داده فعلی

فایل خام اولیه (`customers_requests.sqlite`) در واقع **دو جدول** را به‌صورت افقی
کنار هم داشت: گزارش اصلی (۵۰ ردیف) + یک زیرجدول کوچک شامل **شماره تماس** برای
۶ نفر. هنگام پاکسازی:
- ۴ نفر (سعیدی، فراهانی، مظفری، پاشاپور) دقیقاً همان رکورد قبلی تشخیص داده شدند
  و فقط شماره تماسشان اضافه شد.
- «آقای آزمون» در دو جدول اطلاعات **متضاد** داشت (نوع متقاضی و اقدامات متفاوت) —
  به‌جای ادغام اشتباه، به‌عنوان **دو رکورد جداگانه** نگه داشته شد.
- «آقای قنبری» مشتری کاملاً جدیدی بود که فقط در زیرجدول تماس وجود داشت — به‌عنوان
  رکورد جدید اضافه شد.

نتیجه: **۵۲ رکورد نهایی**، که تنها **۳ نفر** شماره تماس ثبت‌شده دارند.
